# Notebook LM 單一國家數據使用指南

## 📋 概述

已為所有 38 個國家/地區創建了獨立的數據文件，每個文件都包含：
- ✅ 完整的國家數據
- ✅ 明確的數據來源標註
- ✅ 數據收集方式說明
- ✅ 數據代表性說明

## 📁 文件結構

```
各國家數據/
├── 國家索引.txt                    # 所有國家的索引列表
├── Statcounter_中國_數據報告.txt
├── Statcounter_台灣_數據報告.txt
├── Statcounter_美國_數據報告.txt
├── ... (共38個國家文件)
└── Notebook_LM_單一國家使用指南.md  # 本文件
```

## 🚀 使用方法

### 方法一：單獨查詢特定國家

1. **選擇要查詢的國家**
   - 打開 `國家索引.txt` 查看所有可用國家
   - 或直接選擇對應的國家文件

2. **在 Notebook LM 中上傳**
   - 打開 Notebook LM
   - 點擊「新增來源」→「上傳文件」
   - 選擇該國家的數據文件（例如：`Statcounter_台灣_數據報告.txt`）

3. **開始提問**
   - 現在可以針對該國家提問
   - Notebook LM 會自動引用數據來源

### 方法二：同時查詢多個國家（比較分析）

1. **選擇多個國家文件**
   - 例如：台灣、中國、美國

2. **在 Notebook LM 中上傳多個來源**
   - 依次上傳每個國家的文件
   - 或一次性選擇多個文件上傳

3. **進行比較分析**
   - 可以問比較類型的問題
   - Notebook LM 會從多個來源整合信息

## 💡 提問範例

### 單一國家查詢

當你只上傳一個國家文件時，可以問：

- **作業系統相關**

  - "台灣用戶使用 Mac 的比例如何？"

  - "Chrome 在台灣的市佔率是多少？"
  - "台灣用戶使用 Safari 的情況如何？"

- **設備相關**
  - "台灣最流行的螢幕解析度是什麼？"
  - "台灣用戶最常使用的手機品牌是什麼？"
  - "台灣的平板電腦使用情況如何？"

- **時間趨勢**
  - "台灣的瀏覽器市佔率在過去一年有什麼變化？"
  - "台灣的作業系統使用趨勢如何？"

### 多國家比較查詢

當你上傳多個國家文件時，可以問：

- "比較台灣、中國和美國的作業系統使用情況"
- "台灣和日本的瀏覽器偏好有什麼差異？"

## ✨ Notebook LM 如何引用來源

當你在 Notebook LM 中提問時，系統會：

1. **自動識別來源**
   - 從文件中的來源信息識別數據來源
   - 在回答中明確標註來源

2. **引用格式範例**
   ```
   根據 Statcounter Global Stats 的數據...
   
   數據來源：Statcounter Global Stats (https://gs.statcounter.com/)
   統計範圍：台灣地區的網頁瀏覽統計
   ```

3. **提供來源連結**
   - Notebook LM 可能會在回答中提供來源連結
   - 方便你進一步查閱原始數據

## ⚠️ 注意事項

1. **數據時效性**
   - 數據範圍：2008-07 至 2026-01
   - 建議定期更新數據

2. **數據代表性**
   - 數據僅代表使用 Statcounter 追蹤的網站訪問情況
   - 不代表該地區所有網站的訪問情況

3. **文件大小**
   - 每個國家文件大小不同
   - 較大國家的文件可能較大（如：歐洲地區、中國）

4. **Notebook LM 限制**
   - 如果文件太大，可能需要等待處理時間
   - 建議一次上傳 3-5 個國家文件進行比較
